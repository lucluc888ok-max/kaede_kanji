# RPGシステム設計書（保留中）

ステータス振り分け＋ボス討伐システムの設計案。実装時にこのMDを参照して開始する。

---

## コンセプト

- コインをステータスに振ることがゲームの目標になる
- 毎日の小ボス討伐でモチベーション維持
- 週次の中ボスで達成感
- 1学期末の大ボスで「クリア」体験

---

## ステータス 6種

| # | 名前 | 絵文字 | バトルでの役割 |
|---|------|--------|---------------|
| 1 | よみ力 | ⚔️ | 基本攻撃ダメージ |
| 2 | かき力 | 🛡️ | 受けるダメージを減らす |
| 3 | きおく | 💨 | 先制攻撃できる確率 |
| 4 | かんがえ | ✨ | 3ターンに1回の魔法攻撃 |
| 5 | やる気 | ❤️ | 最大HP |
| 6 | ひらめき | 🍀 | クリティカル発生率 |

## バトル計算式

```
自分のHP    = 100 + やる気 × 15
自分のATK   = よみ力 × 5
先制判定    = きおく > random(0〜20) → プレイヤー先制
魔法ダメージ = 3ターンに1回、かんがえ × 8
クリティカル = random(0〜100) < ひらめき → ダメージ×2
受けるDMG   = ボスATK − かき力 × 3（最低 5）
```

## コイン収支の注意点

現在の設定（正解+10、不正解−20）でのコイン収支：

| 20問中の正解数 | 1クイズあたりの収支 |
|--------------|-------------------|
| 16問 | +40 |
| 14問 | +20 |
| 13問 | +10（損益分岐ライン） |
| 12問 | −40 |

**実装前に検討すべき選択肢：**

A. コイン収支を見直す（例：正解+15、不正解−10）
B. バトル専用「バトルポイント（BP）」をコインとは別軸で設ける
   - BPはクイズ完了ごとに付与（正答率に応じて1〜5BP）
   - ステータスはBPで上げる、コインはショップ用に残す
C. 現状のコイン設定のままボス難易度を調整する

```ts

  yomi: number    // よみ力

  kangae: number  // かんがえ
  yaruki: number  // やる気
  hirameki: number // ひらめき

  weeklyBossDefeatedAt: string | null // 今週討伐済みか
  bigBossDefeated: boolean
}
```
